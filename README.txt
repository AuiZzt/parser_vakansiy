# Агрегатор вакансий

Программа для автоматического сбора вакансий с ведущих сайтов по поиску работы: Habr Career, SuperJob и Работа.ру. Все данные собираются в единый JSON-файл и отображаются в удобном веб-интерфейсе.

## Структура проекта

aggregator/
├── manager1.py          # Главный менеджер (управляет всем процессом)
├── main.py              # Основной скрипт парсинга
├── parser/              # Папка с парсерами
│   ├── __init__.py      # Инициализация пакета
│   ├── base.py          # Базовые функции (заголовки, утилиты)
│   ├── habr.py          # Парсер Habr Career
│   ├── superjob.py      # Парсер SuperJob
│   └── rabota.py        # Парсер Работа.ру
├── index.html           # Веб-интерфейс для просмотра вакансий
├── style.css            # Стили для веб-интерфейса
├── script.js            # Логика веб-интерфейса
├── vacancies.json       # Файл с собранными вакансиями
└── requirements.txt     # Зависимости Python

## Системные требования

- Python 3.8 или выше
- Браузер (Chrome/Firefox/Edge) для просмотра результатов
- Интернет для парсинга вакансий

## Запуск программы

### Подготовка

1. Нажмите Code -> Download ZIP
2. Распакуйте архив на Рабочий стол
3. Переименуйте папку в aggregator (если нужно)

### Для Linux/macOS

Перейдите в папку с программой:
cd ~/Desktop/aggregator

Запустите менеджер:
python3 manager1.py

### Для Windows

Перейдите в папку с программой:
cd C:\Users\%USERNAME%\Desktop\aggregator

Если папка находится в OneDrive, используйте:
cd C:\Users\%USERNAME%\OneDrive\Desktop\aggregator

Запустите менеджер:
python manager1.py
или
py manager1.py

## Функции менеджера

После запуска появится меню с выбором действий:

Команда 1 - Установка зависимостей
Создаёт виртуальное окружение (venv), если его нет. Устанавливает все библиотеки из requirements.txt. При необходимости пересоздаёт venv заново.

Команда 2 - Запуск парсинга
Запускает сбор вакансий с сайтов. Автоматически открывает веб-страницу index.html с результатами после завершения.

Команда 3 - Завершить работу
Останавливает все процессы, закрывает сервер и завершает работу программы.

Команда 4 - Проверка статуса
Показывает текущее состояние: сколько вакансий собрано, активен ли сервер, есть ли ошибки.

Команда 5 - Выход из программы
Корректно завершает работу программы и закрывает все процессы.

## Пошаговая инструкция использования

1. Запустите менеджер командой python manager1.py

2. Введите 1 - менеджер создаст venv (если нужно) и установит все зависимости
   - Дождитесь завершения установки
   - Нажмите Enter для продолжения

3. Введите 2 - начнётся парсинг вакансий
   - Дождитесь окончания парсинга
   - Веб-страница index.html откроется автоматически

4. Пользуйтесь сайтом для просмотра вакансий

5. Введите 3 - чтобы завершить работу и закрыть сайт

6. Введите 4 - чтобы проверить статус в любой момент

7. Введите 5 - для выхода из программы

## Как это работает

### Установка зависимостей (команда 1)

- Проверяет наличие папки venv/
- Если нет - создаёт виртуальное окружение
- Если есть - проверяет целостность
- Устанавливает все библиотеки из requirements.txt
- При необходимости пересоздаёт venv заново

### Парсинг вакансий (команда 2)

- Запускает парсеры для всех сайтов
- Собирает вакансии в единый JSON-файл
- Автоматически открывает index.html в браузере
- Показывает прогресс в консоли

### Завершение работы (команда 3)

- Останавливает сервер
- Закрывает браузер
- Завершает все процессы

### Проверка статуса (команда 4)

- Показывает количество собранных вакансий
- Статус сервера (активен/неактивен)
- Состояние парсеров

## Возможные проблемы и решения

### Ошибка "python not found"

Убедитесь, что Python установлен. Скачайте с python.org

### Ошибка с OneDrive на Windows

Используйте альтернативный путь:
cd C:\Users\%USERNAME%\OneDrive\Desktop\aggregator

### Ошибка при создании виртуального окружения

Убедитесь, что у вас есть права на запись в папку. Попробуйте запустить терминал от имени администратора (Windows). Для Linux/macOS: sudo python3 manager1.py

### Ошибка при установке зависимостей

Проверьте подключение к интернету. Попробуйте обновить pip: python -m pip install --upgrade pip

### Менеджер не видит файл requirements.txt

Убедитесь, что файл находится в корневой папке проекта.

## Как работает управление зависимостями

Алгоритм работы команды 1:

1. Проверка venv - если папка venv/ отсутствует, создаётся
2. Проверка целостности - если venv повреждён, пересоздаётся
3. Активация окружения - автоматическая активация виртуального окружения
4. Установка зависимостей - pip install -r requirements.txt
5. Обновление - при необходимости библиотеки обновляются до актуальных версий
